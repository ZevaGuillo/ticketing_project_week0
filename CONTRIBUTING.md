Guía rápida de contribución
Gracias por contribuir al proyecto Ticketing Project.

Si revisas el repositorio y quieres seguir la documentación centralizada, usa estos enlaces rápidos:

- **AI Workflow** — [AI_WORKFLOW.md](AI_WORKFLOW.md)
- **Human Checks** — [humanchcks.md](humanchcks.md)
- **Dept Report** — [deptReport.md](deptReport.md)

Guía rápida de contribución

1. Crea un fork o una rama a partir de `main`.
2. Realiza cambios pequeños y atómicos. Incluye tests cuando apliquen.
3. Abre un Pull Request con descripción clara y referencias a `specs/` o `tasks.md` si aplica.

Comunicación

- Usa Issues para reportar bugs o proponer features.
- Etiqueta claramente los issues con `bug`, `enhancement`, `docs`, `question`.

Gracias por colaborar.
